A versão 3.8.0 do LAND IT encontra-se agora disponível. Segue-se a lista de alterações e novas funcionalidades.

<!-- truncate -->

-    Cada unidade de transformação agora tem um ID específico quando associada a uma versão
-    Adição de uma base de conhecimento para as operações, de modo a estabelecer qual a sequência de operações correta para uma determinada transformação em certas características do território
-    Adição de variáveis de execução de modo a justificar a utilização de uma sequência de operações diferente para uma mesma transformação com as mesmas características do território: socalcos, subprodução e replantar
-    Adição do shapefile da POSA à exportação de uma versão, visto que agora a POSA pode sofrer atualizações
-    Correção de bugs:
     -    Ao exportar um cenário a indicação de mosaicos agora dá-se pela sigla AEMGC;
     -    Ao importar um cenário as atribuições de declive e estrutura de resiliência não estavam a ser guardadas
     -    Ao duplicar um cenário as transformações associadas às operações complementares não estavam a ser duplicadas

<details>
<summary>Mostrar detalhes</summary>

-    Ao importar um cenário, o ID de uma unidade de transformação na respetiva versão é proveniente do shapefile das UIs do campo UI_c;
-    A base de conhecimento pode ser visualizada no menu de Edição de Transformações, e ao escolher uma transformação para um polígono, as transformações pertencentes à base de conhecimento estarão realçadas;
-    Após uma atualização da POSA, o utilizador pode recorrer à base de conhecimento para atribuir transformações aos polígonos em que ocorreram alterações do valor da POSA.

</details>
