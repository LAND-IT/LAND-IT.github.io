LAND IT version 3.8.0 is now available. Below is the list of changes and new features.

<!-- truncate -->

-    Each transformation unit now has a specific ID when associated with a version
-    Addition of a knowledge base for operations, in order to establish the correct sequence of operations for a given transformation in certain territory characteristics
-    Addition of execution variables to justify the use of a different sequence of operations for the same transformation with the same territory characteristics: terraces, underproduction and replant
-    Addition of the POSA shapefile to the export of a version, since now the POSA can undergo updates
-    Bug fixes:
     -    When exporting a scenario, the indication of mosaics is now given by the acronym AEMGC;
     -    When importing a scenario, the slope and resilience structure assignments were not being saved
     -    When duplicating a scenario, the transformations associated with complementary operations were not being duplicated

<details>
<summary>Show details</summary>

-    When importing a scenario, the ID of a transformation unit in the respective version comes from the UI shapefile of the UI_c field;
-    The knowledge base can be viewed in the Transformation Editing menu, and when choosing a transformation for a polygon, the transformations belonging to the knowledge base will be highlighted;
-    After a POSA update, the user can use the knowledge base to assign transformations to polygons where POSA value changes occurred.

</details>
